# 开发规范：地形可视化大改（中间低频 / 周围高频 / 像真正的地形）

- 触发：用户反馈「地形可视化大改，一点都不好看，要中间低频周围高频，像真正的地形」
- 目标文件：`frontend/src/components/Visualizer3D.jsx`（仅 `ocean` 地形模式相关代码）
- 关联：`docs/AI_PROJECT_GUIDELINES.md`、`docs/DEV_VISUALIZER_PERF_GALAXY.md`

## 现状（修改前）

| 项 | 现状 | 问题 |
|---|---|---|
| 频段映射 | `terrainBand = floor((1 - rN) * 63)` → **中心=高频、外圈=低频** | 与用户要求「中间低频、周围高频」**正好相反** |
| 基础地形 | `idleH = sin*0.02 + cos*0.015`，几乎平地 | 不像山，无起伏感 |
| 音频驱动 | 高度由对应频段能量驱动，增益 `(1 - rN*0.35)` | 中心高频权重反而更高 |
| 着色 | 低谷深 / 中坡 accent / 峰泛白，无等高线 | 点云平淡，缺乏地形图质感 |
| 形态 | 点云（Points）铺在 XZ 平面，高度在 Y | 渲染观感没问题，需内容更像地形 |

## 设计

### 目标效果

真正「地形」观感：
1. **中心=低频（大尺度主峰）**，低频能量强 → 中心隆起高耸的主山脉
2. **外围=高频（细密细节）**，高频能量弱 → 外围细碎小起伏的纹理
3. 即使无音乐，也显示**静态 fBm 山脉**（中心高、边缘沉入的悬浮山岛）
4. 着色用**地形色带 + 等高线**：谷底深色 → 坡面 accent → 峰顶泛白，每隔固定高度一条亮等高线
5. 音频在静态山基础上叠加起伏，低频中心振幅大、高频外围振幅小

### 改动 1：`buildBase` 地形基础（第 380-394 行）

- **频段映射反转**：`terrainBand = floor(rN * 63)` → 中心(0)=低频，外圈(63)=高频
- **静态 fBm 山脉**：用 4 个八度正弦叠加（确定性随机相位 `trPhase/trFreq`，`mulberry32(0x51ed270b)` 生成，重建一致），`radial = pow(1-rN, 1.5)` 让中心高、边缘沉入
- 基础高度存入 `basePositionsTerrain[idx*3+1]`

### 改动 2：动画 `ocean` 分支（第 940-972 行）

- `freqWeight = (1 - rN)`：中心(低频)权重高、外圈(高频)权重低
- `audioH = energy * planeSize * TERRAIN_GAIN * (0.35 + freqWeight * 1.9) * (1 + beatFreqBoost*1.8)`：低频大幅、高频小幅
- 叠加静态山体 `by`、鼓点脉冲（中心更强）、中心涟漪、待机呼吸
- 振幅 clamp 到 `[-0.28, 0.52] * planeSize`

### 改动 3：着色 `isTerrain` 分支（第 1026-1062 行）

- 地形色带：谷底深（accent*0.06）→ 坡面（accent*0.72）→ 峰顶泛白（crest*0.5）
- 谷底压暗增加纵深（`vally` 因子）
- **等高线**：`fract(normH * 11)` 接近 0.5 边缘时画亮线，模拟地形图
- 中心主峰(低频) `centerGlow = (1-rN)*0.22` 更亮
- 音频能量点亮

### 不改动的部分

- `galaxy` / `liquidmetal` / `coverflow` 三形态逻辑不变
- 手势、相机、过渡动画不变
- 节拍检测、冲击波、鼓点频率增强机制不变
- 频谱来源 `getSpectrumBars` 不变（0=低频 63=高频，已确认）
- 常量 `TERRAIN_SIZE_RATIO=1.35`、`TERRAIN_GAIN=0.26` 复用（新增 `trPhase/trFreq` 局部参数）

## 验收标准

- [ ] 地形中心对应低频、外围对应高频（与用户描述一致）
- [ ] 无音乐时显示静态山脉（中心高、边缘低的山岛），非平地
- [ ] 播放时低频在中心形成大起伏、高频在外围形成细节
- [ ] 着色有地形色带 + 等高线，像地形图
- [ ] 三形态切换（galaxy/liquidmetal/coverflow）零回归
- [ ] `vite build` 编译通过；CI 构建 APK 成功

## 风险与回退

- 纯听觉可视化参数调整，无运行时崩溃风险
- 若山脉过陡/过平，调 `TERRAIN_GAIN` 与基础高度 `*0.38` 系数即可
- 回退：保留旧逻辑可随时还原
