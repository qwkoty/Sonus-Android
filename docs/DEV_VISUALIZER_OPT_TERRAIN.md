# 开发规范：地形（ocean）可视化优化

- 触发：用户要求分析并优化「地形」可视化（上一轮已大改频段映射与色带/等高线，本轮做"像真正的地形"的深化）
- 目标文件：`frontend/src/components/Visualizer3D.jsx`（terrain 动画/着色）
- 关联：`docs/DEV_VISUALIZER_TERRAIN.md`（上一轮改动）、`docs/AI_PROJECT_GUIDELINES.md`

## 现状（修改前，即上一轮结果）

| 模块 | 现状 | 代码定位 |
|---|---|---|
| 基础形态 | 极坐标 fBm 山脉，中心低频/外圈高频，悬浮山岛 | 384-404 |
| 动画 | 静态山体 + 低频(中心)大幅/高频(外围)小幅 + 鼓点脉冲 + 中心涟漪 + 待机呼吸 | 950-983 |
| 着色 | 地形色带(谷底深→坡 accent→峰泛白) + 等高线 + 中心主峰亮 | 1037-1084 |

## 问题分析

1. **点云无表面法线 → 不像"真正的地形"**：当前是 19881 个独立点，每个点只按自身高度着色，没有光照，因此看起来像"漂浮的彩色点阵"，而非有体积的地形。这是"像真正的地形"最核心的缺失。
2. **起伏无立体明暗**：山峰朝光面、背风面、谷地都同样亮度，读不出地形起伏的体量感。
3. **缺少大气/纵深**：远处与高处无雾/渐变，画面偏平。
4. **顶点着色与音频脱节**：着色用归一化高度 `normH` 驱动色带，但没结合"光照方向"，所以音频起伏不会产生明暗变化。

## 优化方案

### 1. 计算法线 + 漫反射光照（像真正地形的关键）

地形是规则 `GRID`(≈141×141) 网格，可用**有限差分**从相邻粒子高度求法线。

- 在 `buildBase` 后（或动画首帧）为每个粒子预存**网格邻居索引**：`terrainNb[i] = {l, r, u, d}`（左/右/上/下在 `COUNT` 数组中的下标，边界用自身）。存为 `Int32Array(COUNT*4)`。
- 动画每帧：先把每个粒子当前高度写入 `terrainH[i]`（已有的 `y` 值），再开**第二趟**遍历，用邻居高度差算法线：
  ```js
  // 邻点水平/垂直步距 ≈ step
  const hl = terrainH[nb_l], hr = terrainH[nb_r], hu = terrainH[nb_u], hd = terrainH[nb_d];
  const dx = (hr - hl), dy = (hd - hu);   // step 已隐含在高度差比例
  // 法向 ≈ normalize(-dx, -dy, 2*step)  （z 向上）
  const nzx = -dx, nzy = -dy, nzz = 2 * step;
  const nl = Math.hypot(nzx, nzy, nzz);
  nx = nzx/nl; ny = nzy/nl; nz = nzz/nl;
  ```
- **虚拟光照**（光从左上偏上打入）：
  ```js
  const lx=-0.5, ly=-0.4, lz=0.75; // 归一化
  const diff = Math.max(0, nx*lx + ny*ly + nz*lz); // 漫反射
  ```
- 着色色带乘以 `(0.35 + diff*0.85)`：朝光坡亮、背风坡与谷地暗 → **地形起伏立刻有体量感**。

### 2. 大气 / 高度雾增强纵深

- 按高度 `normH` 或到中心距离 `rN` 做轻度雾：高处/远处叠加一层背景主题色微光，拉开层次。
- 可选：底部谷地偏冷暗、峰顶偏暖亮（与色带叠加），强化"日照地形"观感。

### 3. 谷底积水反光（可选点缀）

- `normH < 0.18` 的低谷区域，叠加一层偏蓝、带高光反射的色调，模拟湖泊/积水，增加"地形"可读性。
- 低成本：在色带 valley 分支里把蓝色权重提高 + 一点 spec。

### 4. 性能

- 第二趟法线遍历是 `O(COUNT)` + 4 次邻居数组读取，约 8 万次内存访问/帧，开销极小（远低于已存在的 20k 粒子主循环）。
- `terrainNb` 邻居索引在 `buildBase` 一次性算好（基于 `gy/gx` 网格坐标直接取 `idx±1 / ±GRID`），无每帧成本。

## 验收标准

- [ ] 地形出现明显的坡面明暗（朝光亮、背光暗），不再是平铺点阵
- [ ] 音频起伏时明暗随高度变化，山体有体量感/立体感
- [ ] 远处/高处有轻度大气雾，纵深增强
- [ ] 帧率稳定（法线第二趟开销可忽略）
- [ ] 三模式切换零回归；`vite build` + CI 通过

## 风险与回退

- 法线有限差分对边界粒子（邻居=自身）会产生退化法向，需 `nl` 防御 0；边界用 `nz=1` 兜底
- 若坡度噪声导致法线过抖，可对 `dx/dy` 做轻度平滑（取更大邻域或降采样）
- 光照方向与色带若冲突，`diff` 权重 0.85 可调
- 回退：删除第二趟法线遍历与 `diff` 调制，回到上一轮色带/等高线版本
