# DEV 开发规范：可视化参数回调 + 鼓点阈值调高（v1.22）

> 适用范围：Sonus 播放器全可视化系统（engine.js / Visualizer.jsx / Visualizer3D.jsx）。
> 文档性质：方案草案（先评审后实施）。
> 触发原因：v1.21 参数叠加过猛——wave 几乎全程满屏、地形山太高、绿色湖面底座无用、鼓点太敏感。
> 关联历史：`DEV_VISUALIZER_OPT_V2.md`(v1.21 方案)。

---

## 0. 问题诊断（基于用户截图与反馈）

| 问题 | 截图表现 | 根因 |
|---|---|---|
| Wave 模式"一直很高很高" | 第一张截图：频谱柱几乎满屏，安静段和高潮段几乎一样高 | engine.js `gain=1.12` + `pow(0.55)` + `smoothing(0.65)` 叠加使整体基线抬升；Visualizer `maxAmp=0.48` 放大后更满 |
| 3D 地形"太高了" | 第二张截图：山峰尖锐且极高 | 山高乘子 `0.95` + `TERRAIN_GAIN=0.30` + clamp `0.95` 三重叠加 |
| "下面那个绿色的底座没啥用" | 第二张截图底部：发光的青色圆盘 | 新增的 `lakeMesh` 发光湖面倒影，实际效果像一块塑料底座 |
| 鼓点打击太敏感 | 冲击波/脉冲频繁触发 | `BEAT_THRESHOLD=0.07` 极低；2D `bass>0.32` 太宽松 |

**核心结论**：v1.21 四个域各自放大了参数，叠加后产生"过曝"效应。本次是**收敛回调**，不是新增功能。

## 1. 域 A — engine.js 音频范围收敛

### 1.1 现状（v1.21 改动后）
```js
analyser.smoothingTimeConstant = 0.65;   // 原 0.75
analyser.minDecibels = -100;              // 原 -90
analyser.maxDecibels = -10;               // 原 -15
const corrected = Math.min(1, Math.pow(normalized, 0.55) * gain); // 原 pow(0.6), gain=1.12
```

### 1.2 具体改动
```js
analyser.smoothingTimeConstant = 0.72;    // 0.65→0.72：折中（比原 0.75 略跟手，但不抖）
// minDecibels / maxDecibels 保持 -100 / -10 不变（已验证合理）
const gain = 1.04;                       // 1.12→1.04：微提亮即可，不过曝
const corrected = Math.min(1, Math.pow(normalized, 0.58) * gain); // 0.55→0.58：稍微压暗一点
```

**理由**：`smoothing=0.65` 在手机上确实偏抖（用户截图未直接体现抖动，但安全起见回调到 0.72）；`gain` 从 1.12 降到 1.04 是关键——这是全局乘数，对 wave/terrain/galaxy 全部生效；`pow` 从 0.55 回到 0.58 让安静段更暗、高潮段对比更强。

## 2. 域 B — 2D 波可视化（`Visualizer.jsx`）回调幅度

### 2.1 参数表

| 参数 | v1.21 值（过高） | v1.22 目标值 | 位置 |
|---|---|---|---|
| `maxAmp`（wave 模式镜像振幅） | `h * 0.48` | `h * 0.36` | 第 ~338 行 |
| `MAX_R`（ring 外圈半径） | `minDim*0.5*0.94` | `minDim*0.5*0.90` | 第 ~133 行 |
| 辐射环振幅 | `(MAX_R-INNER_R)*0.20` | `*0.15` | 第 ~253 行 |
| 中心填充波纹 | `layerValue*minDim*0.06` | `*0.045` | 第 ~155 行 |
| breathScale bass 项 | `bassSmooth*0.20` | `*0.14` | 第 ~130 行 |
| 冲击波触发阈值 | `bass>0.32 && bassDelta>0.08` | `bass>0.45 && bassDelta>0.14` | 第 ~149 行 |
| 频谱尖刺长度 | `v*minDim*0.06` | `*0.04` | 尖刺段 |

### 2.2 效果预期
- Wave 模式：安静段落频谱柱高度 ≈ 15–25% 屏幕高度；鼓点/高潮段冲到 ≈ 60–80% → **有明显呼吸感**
- Ring 模式：外圈不再贴边（留 10% 边距），辐射环振幅收窄
- 冲击波只在真正的低音重击时出现（bass > 0.45 且上升沿 > 0.14），不再每帧都冒泡

## 4. 域 D — 全局鼓点阈值调高（所有可视化模式统一）

### 4.1 核心鼓点检测（3D 主循环，影响 galaxy/liquidmetal/ocean/coverflow 全部模式）

| 参数 | 当前值（太敏感） | v1.22 目标值 | 影响 |
|---|---|---|---|
| `BEAT_THRESHOLD` | `0.07` | `0.12`（+71%） | 低频导数必须更大才判定为"一拍" |
| `BEAT_COOLDOWN` | `0.05s` | `0.08s`（+60%） | 两拍之间最少间隔更长，避免连击 |
| `SHOCK_GAIN` | `2.6` | `2.2`（-15%） | 鼓点冲击力稍减 |

### 4.2 各模式专属鼓点响应

| 位置 | 当前 | v1.22 | 说明 |
|---|---|---|---|
| 2D shockwave 触发 | `bass>0.32 && bassDelta>0.08` | `bass>0.45 && bassDelta>0.14` | 只在真·重低音时出波 |
| Clay 脱落频谱触发 | `CLAY_SPECTRUM_TRIGGER=0.55` | `0.62` | 封面粒子脱落更难触发 |
| Liquidmetal bassBoost | `planeSize*0.26` | `*0.20` | 液态金属球鼓点膨胀收敛 |
| Galaxy bassPush（核球推挤） | `planeSize*0.18` | `*0.13` | 星系核球呼吸幅度收敛 |
| Terrain beatPulseH | `planeSize*0.22` | `*0.14`（已在 3.2 表） | 地形鼓点抬升收敛 |
| Terrain ripple（涟漪） | `beatPulseRef*planeSize*0.07` | `*0.045` | 地形涟漪幅度收敛 |

### 4.3 效果预期
- 安静段落（人声、吉他Solo）：几乎没有冲击波/脉冲/涟漪，可视化平稳
- 鼓点段落（Drop、Kick）：清晰的单次脉冲 + 适度涟漪，不连发
- 用户感受从"一直在跳"变成"该跳的时候才跳"

## 6. 验收标准

- [ ] Wave 模式：**安静时频谱柱 ≤ 25% 屏幕高度**，高潮/鼓点时 ≤ 75%，有明显呼吸层次
- [ ] Ring 模式：外圈不贴边，有 10%+ 黑边余量
- [ ] 3D 地形：山峰高度适中（不超过画面 60%），无绿色底座圆盘
- [ ] 鼓点冲击波：安静音乐（如周杰伦《晴天》主歌）几乎不出波；Drop 段出 1–2 个波
- [ ] 灵气光点可见但不抢眼（半透明漂浮感）
- [ ] `vite build` 通过，无报错

---

## 7. 风险控制

- 本次全是**数值回调**，不涉及结构变更，风险极低
- 若回调过度（wave 太矮/地形太平），可微调 `gain` 或各乘子
- `lakeMesh` 删除为不可逆操作（删代码），若后续想要类似效果可用其他方式实现（如地面渐变雾气）
