# DEV 开发规范：移除滑动切模式 + 恢复 360° 拖拽旋转（v1.28）

> 适用：Sonus 播放器可视化交互（Player.jsx、Visualizer3D.jsx）。
> 原则：最小改动、不破坏已有功能、不触碰视觉设置面板的按钮切模式。

---

## 一、需求

1. **移除"滑动屏幕切换可视化"功能**：当前横向快扫屏幕会从当前可视化模式切到下一个/上一个模式，删除该手势。
2. **恢复 360° 旋转**：用户反馈原本的 360° 旋转（单指/鼠标拖拽让 3D 画面绕任意方向旋转）被上一轮"优化"去掉了，需要恢复。

---

## 二、现状与根因分析

### 2.1 滑动切模式（待删除）

- 位置：`frontend/src/pages/Player.jsx` 267–288 行。
- 实现：`swipeRef` + `onVizTouchStart` + `onVizTouchEnd`，判定"横向快扫"（|dx|>60 且 |dx|>|dy|×1.5 且 dt<400ms）后 `setVm` 切模式并 `buzz(12)`。
- 绑定：`Player.jsx` 308 行背景层 `<div ... onTouchStart={onVizTouchStart} onTouchEnd={onVizTouchEnd}>`。

### 2.2 360° 旋转（待恢复）

- 位置：`frontend/src/components/Visualizer3D.jsx`。
- 关键代码：
  - 130–150 行 `gestureRef`：偏航 `rotationY`（360°）+ 俯仰 `rotationX`（±85°）、`dragging` / `pinching` 状态、`autoRotate` 默认 `false`。
  - 1291–1311 行：每帧把 `g.rotationY/X` 缓动后写入 `points.rotation.y/x`，实现拖拽旋转。
  - 1354–1436 行：canvas 上的 `touchstart/touchmove/touchend`（单指拖拽旋转、双指捏合缩放+扭转）、`wheel`（滚轮缩放）、鼠标 `mousedown/move/up`（桌面拖拽同单指）。
- **结论**：360° 手动拖拽旋转的代码**目前仍然存在且完整**，并未被真正删除。它"失效"的根因是 **2.1 的滑动切模式手势与其抢占了同一横向手势**——用户在 3D 画面上横滑想旋转时，快扫被 `onVizTouchEnd` 判定为"切模式"，旋转手势被吞掉，体感即"360° 旋转没了"。
- 因此：**删除滑动切模式后，360° 拖拽旋转会自动恢复，无需再为旋转新增代码。**

### 2.3 不改动项（避免误伤）

- `buzz()`（Player.jsx 264–265 行）：被播放/上一首/下一首、视觉设置面板按钮使用，**保留**。
- `switchVm()`（Player.jsx 289 行）：被"视觉设置面板"的模式按钮（510 行）使用，**保留**。即"按钮切模式"保留，仅删除"滑动切模式"。
- Visualizer3D.jsx 的旋转手势代码：保留（正是要恢复的能力）。

---

## 三、修改方案

### 改动 1：Player.jsx 删除滑动切模式

删除 267–288 行整段（从 `// 横向滑动切换可视化模式` 注释到 `onVizTouchEnd` 的闭括号），包括：
- `swipeRef` useRef
- `onVizTouchStart`
- `onVizTouchEnd`

### 改动 2：Player.jsx 解除背景层手势绑定

将 308 行：
```jsx
{/* 可视化背景层（承载横向滑动切模式手势） */}
<div style={{ position: 'absolute', inset: 0 }} onTouchStart={onVizTouchStart} onTouchEnd={onVizTouchEnd}>
```
改为（移除 touch 绑定与过时注释）：
```jsx
{/* 可视化背景层 */}
<div style={{ position: 'absolute', inset: 0 }}>
```

### 改动 3（无代码）：360° 旋转恢复

移除滑动手势后，Visualizer3D 的 canvas 触摸/鼠标监听不再被上层快扫拦截，单指/鼠标拖拽即可 360° 偏航+俯仰、双指缩放扭转、滚轮缩放——能力恢复。

> 备注：自动旋转/自转（`autoRotate` 默认 `false`，此前按"永远不转"偏好关闭）属另一项独立能力，本次**不启用**，保持现状。如需开启，另行提 DEV 规范。

## 四、影响范围

| 项 | 影响 |
|---|---|
| 滑动切模式 | 删除，横滑不再切模式 |
| 视觉设置面板按钮切模式 | 不受影响（仍可用） |
| 360° 拖拽旋转 | 恢复（此前被滑动手势吞掉） |
| 播放/上一首/下一首/调色等 | 不受影响（`buzz` 保留） |
| 2D/3D 各可视化模式渲染 | 不受影响 |
| 歌词、搜索、队列等 | 不受影响 |
