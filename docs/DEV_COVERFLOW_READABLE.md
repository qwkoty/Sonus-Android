# 开发规范：粒子封面「脱落仍清晰可读」修正

- 触发：v1.13 腻子脱落动画用户满意"落下去"的动效，但反馈**封面不完整、看不清**。
  要求：**就算粒子落下去，封面也必须清晰可辨认；且落了之后会自动浮回**。
- 目标文件：`frontend/src/components/Visualizer3D.jsx`
- 关联：`DEV_COVERFLOW_CLAYFALL.md`（v1.13 脱落机制）

## 一、现状诊断

| 问题 | 当前代码 | 视觉后果 |
|---|---|---|
| 脱落即"变色" | `applyCoverColors`：`li = 1 - cd*0.75`（深度1→亮度0.25）、`am = cd*0.9`（深度1→90%主题色） | 脱落粒子变成暗淡的主题色噪点，封面图案被"洗掉" |
| 待机也变暗 | 待机着色：`layerDim = 1 - clayDepth*0.75` | 无封面时脱落粒子暗 75%，整体发灰 |
| 脱落位移过大 | `layerZ = cd * planeSize * LAYER_GAP * (COVER_LAYERS-1)` = `cd * 0.30 * planeSize` | 脱落粒子被推后 ~30% 画幅，透视错位形成"破洞"感 |

核心结论：**封面"看不清/不完整"不是因为粒子少了，而是脱落粒子被改了颜色 + 推得太远**。
每个 (u,v) 网格仍由唯一粒子占据，只要该粒子**保持封面色与亮度**，封面就始终可读 ——
脱落只应表现为"这块腻子轻微凹陷/浮起"的立体呼吸，而非变色破洞。

## 二、修正方案

### 1) 脱落粒子保持封面色（关键）

`applyCoverColors` 中，深度只用于**极轻微的明暗/微染色**，不再大面积替换成主题色：

```js
const cd = clayDepth[i];                 // [0,1]
const li = 1.0 - cd * 0.20;             // 深度1→亮度0.80（仍清晰）
const am = cd * 0.15;                   // 深度1→仅15%主题色微染（立体感，不洗图案）
// 其余不变：封面像素 s[] 仍占主导，脱落粒子依旧是该位置的专辑色
```

### 2) 待机着色同样只轻微变暗

```js
const layerDim = 1.0 - clayDepth[i] * 0.20; // 深度1→亮度0.80（原0.25）
```

### 3) 收敛 Z 位移，消除"破洞"

新增系数把最大凹陷从 `0.30*planeSize` 收到约 `0.18*planeSize`，既保留可辨的"掉落"动效，
又避免透视错位造成封面破碎感：

```js
const CLAY_FALL_DEPTH_SCALE = 0.6;      // 凹陷深度缩放（1=原0.30画幅，0.6≈0.18画幅）
...
const layerZ = cd * planeSize * LAYER_GAP * (COVER_LAYERS - 1) * CLAY_FALL_DEPTH_SCALE;
```

## 三、保持不变

- 脱落机制本身（v1.13）：随机位置、≈1500~2000 同时起落、`MAX_CLAY_FALLEN` 上限、自动回弹 —— 全部保留。
- 用户已满意的"落下去"动效观感保留，仅做清晰度修正。

## 四、验收标准

- [ ] 大量粒子脱落时，封面图案/文字**依旧清晰可辨认**（不再被洗成主题色噪点）
- [ ] 脱落粒子只是"轻微凹陷/浮起"的立体呼吸，不出现破洞
- [ ] 脱落粒子会**自动浮回前层**，随后又有新的落下去（循环不变）
- [ ] 待机（无封面）时脱落粒子仍清晰可见、不发灰
- [ ] `vite build` 通过

## 五、可调参数

| 参数 | 位置 | 调大效果 | 调小效果 |
|---|---|---|---|
| `CLAY_FALL_DEPTH_SCALE` | clay 常量区 | 脱落更"深"、更明显，但易显破洞 | 更平、更像微呼吸 |
| `li` 中 `cd*0.20` 系数 | `applyCoverColors` | 脱落更暗 | 脱落更亮、最清晰 |
| `am` 中 `cd*0.15` 系数 | `applyCoverColors` | 脱落更偏主题色 | 脱落越接近纯封面色 |
