# 开发规范：星空漩涡（galaxy）可视化优化

- 触发：用户要求分析并优化「星空漩涡」可视化
- 目标文件：`frontend/src/components/Visualizer3D.jsx`（galaxy 基础/动画/着色）
- 关联：`docs/DEV_VISUALIZER_PERF_GALAXY.md`、`docs/AI_PROJECT_GUIDELINES.md`

## 现状（修改前）

| 模块 | 现状 | 代码定位 |
|---|---|---|
| 基础形态 | 对数螺旋星系盘 + 核球 + 弥散晕，5 旋臂，内窄外宽 | 319-382 |
| 动画 | 差速旋转(内快外慢) + 鼓点切向冲击 + 频谱抖动 + 整体呼吸 + 核心心跳 + 流星 | 760-865 |
| 着色 | 内核=主题色(炽热)、外圈=封面平均色(冷)、晕偏冷暗、星云漂移、闪烁、流星增亮 | 989-1036 |

## 问题分析

1. **核心不够"亮核"**：核球虽有心跳与额外亮度(+0.5)，但缺乏柔和的"核心辉光"渐变，星系中心没有那种恒星密集的眩光感。
2. **旋臂缺乏"密度波"呼吸**：真实星系的旋臂是**密度波**——亮度会沿臂随时间缓慢推进。当前旋臂亮度主要靠静态 `armWave`，没有"波峰沿臂移动"的观感。
3. **角色差异化不足**：核球/盘面/晕都用统一 `PointsMaterial` 大小，盘面粒子可以更细、核球更亮更大，纵深更强。
4. **性能**：差速旋转每粒子 `cos/sin(ang)` 必要但占开销；流星生成每帧最多扫 60 次随机，可优化为预选区。
5. **色彩纵深**：旋臂与外圈晕色相未明显分离，远处纵深感弱。

## 优化方案

### 1. 核心辉光增强

在着色分支对核球粒子叠加**软核辉光**（按到中心距离的二次衰减）：

```js
if (galaxyBulge[i]) {
  const core = Math.pow(1 - rN, 2.2);          // 越靠中心越亮
  intensity += 0.6 + bassAttack * 1.0 + core * 0.8;
}
```
并对整体核心区叠加一层"辉光 halo"：半径 `< 0.12*Rmax` 的粒子额外 +`core*0.4`，形成眩光。

### 2. 旋臂密度波（关键观感提升）

新增随时间的**螺旋密度波亮度**：让旋臂的亮峰沿角向缓慢推进，模拟真实星系密度波。

```js
// 在动画/着色复用 galaxyArm[i] 与 galaxyTheta[i]
const waveAng = galaxyTheta[i] + time * 0.25;        // 波沿臂推进
const densityWave = 0.5 + 0.5 * Math.sin(waveAng * GALAXY_ARMS - time * 0.6);
if (galaxyArm[i] >= 0) intensity *= (0.7 + densityWave * 0.6); // 仅旋臂粒子受密度波调制
```
注意：`densityWave` 在着色分支按 `galaxyArm[i]` 判断是否旋臂粒子，使旋臂出现"亮带绕中心流动"的效果。

### 3. 按角色差异化尺寸与亮度

- 在 `buildBase` 或首帧，给 `galaxyArm`/`galaxyBulge` 粒子设置**尺寸系数**：核球稍大更亮、盘面标准、晕更小更暗。
- 实现：扩充 `PointsMaterial` 无法逐粒子尺寸，需改用 **`size` attribute + 自定义 ShaderMaterial** 或 `Points` 的 per-point size（three.js 原生 PointsMaterial 不支持逐点 size，需用 `onBeforeCompile` 注入或 `ShaderMaterial`）。**低成本替代**：仅通过着色 `intensity` 差异体现（晕 `*0.55` 已有，加大到 `*0.45`，核球 `+0.6`），暂不引入 ShaderMaterial 以控制风险。

### 4. 色彩纵深（旋臂 vs 晕 色相分离）

- 旋臂粒子：核心色→封面色的现有插值保持。
- 晕粒子：在当前"偏冷偏暗"基础上，把色相向**更蓝/更紫**偏移（`b` 增、`r` 减），强化与旋臂暖核的对比，纵深更明显。

### 5. 性能优化

- **差速旋转**：`ang = time*(innerOmega + (outerOmega-innerOmega)*rN)` 的 `cos/sin` 每帧必要，保留。可考虑将 `innerOmega/outerOmega` 与 `rN` 的线性组合在循环外预乘系数，略减乘法，收益小，可选。
- **流星生成**：把"扫 60 次随机找 R>0.45Rmax 粒子"改为预生成一批候选索引（buildBase 时收集 `rN>0.45 && !bulge` 的索引池），生成时直接池内随机取，省去每帧扫描。

## 验收标准

- [ ] 星系中心出现柔和眩光核，比当前更"恒星密集"
- [ ] 旋臂有沿臂流动的亮带（密度波），而非静态起伏
- [ ] 核球/盘面/晕亮度层次更清晰，纵深更强
- [ ] 流星生成不再每帧随机扫描（性能稳）
- [ ] 三模式切换零回归；`vite build` + CI 通过

## 风险与回退

- 密度波若太快显乱，调 `time*0.25` / `time*0.6` 系数即可
- 若引入 ShaderMaterial 做逐点尺寸，风险升高；本期**优先只改着色 intensity**，不引入新材质
- 回退：删除密度波/核心辉光增量即回原状
